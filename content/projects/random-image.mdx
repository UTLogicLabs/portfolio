A local development tool for serving random placeholder images by dimensions — the Lorem Picsum experience, running entirely on your own machine against your own images.

## Background

During my internship at The Church of Jesus Christ of Latter-day Saints, the team was building UI mockups and needed realistic placeholder images without depending on an external service. Lorem Picsum was the obvious inspiration, but we needed something that worked offline and pulled from our own image library.

I built the original version during that internship, then came back to it later and rewrote it with cleaner code — same Node.js and Express stack, same URL-based interface, just a much better implementation.

## Setup

The server expects your images to live in a folder named `images` one level above the project root (`../images`). Drop your images there, start the server, and the API is ready.

```
your-workspace/
├── images/        ← your image library goes here
└── randomImage/   ← the project
```

## How It Works

Requests are structured around width and height path segments. Hit `/200/300` and you get back a 200×300 image randomly selected from your local image folder. Every request is stateless — no auth, no sessions, just a URL that returns an image.

What sets it apart from simply pointing a static file server at a folder is the full API surface: seed-based determinism for consistent results across renders, direct image targeting by ID, and on-the-fly image effects. You can drop these URLs anywhere in a mockup and get predictable, realistic images every time.

To get consistent results across requests, pass a `seed` value and the same image will always be returned for that seed. To target a specific image, swap in an `id` instead.

## API Reference

| Endpoint                           | Description                          |
| ---------------------------------- | ------------------------------------ |
| `/:width/:height`                  | Random image at the given dimensions |
| `/image/id/:id/:width/:height`     | Specific image by ID                 |
| `/image/seed/:seed/:width/:height` | Deterministic random image by seed   |

**Query parameters** work on any endpoint:

- `?grayscale` — convert the image to black and white
- `?blur=1–10` — apply a blur effect at the given intensity

Example: `/image/id/42/400/300?grayscale&blur=2` returns image 42 at 400×300, grayscale with a light blur.
